Lesson 5.4 — Compliance & Policy Awareness

Acceptable vs. Unacceptable AI Use: A Scenario-Based Guide for Tech Professionals

Knowing the rules matters. Knowing how to apply them in real situations is the actual skill. Work through these ten workplace scenarios and test your judgment before you need it on the job.

For each scenario below, decide whether the AI use described is Acceptable, Unacceptable, or a Gray Area requiring more information or caution. Read the explanation after you choose — the reasoning matters as much as the verdict.

Scenario 1 of 10
Using AI to draft a cold outreach email to potential enterprise clients
A sales engineer uses an approved AI tool to draft prospecting emails to companies they've never worked with. The email describes the company's product capabilities. No client data is involved — just publicly available information about the target companies.
Is this acceptable, unacceptable, or a gray area?
Scenario 2 of 10
Pasting a client's database schema into a free AI tool to get help with a query optimization problem
A backend developer is having trouble with a slow SQL query on the client's system. They paste the database schema (table names, column names, and relationships) into ChatGPT's free tier to get optimization suggestions. The schema doesn't contain actual data — just the structure.
Is this acceptable, unacceptable, or a gray area?
Scenario 3 of 10
Using AI to generate test data that mimics the format of real user records
A QA engineer needs realistic-looking test data to populate a staging environment. They use an AI tool to generate synthetic names, emails, and transaction records that follow the same format as real data — but with completely fictional entries. No real user data is used in the prompt.
Is this acceptable, unacceptable, or a gray area?
Scenario 4 of 10
Using AI to write a performance review for a direct report, including their actual performance history
A manager pastes a summary of an employee's quarterly performance — specific projects, ratings, and areas for improvement — into a free AI tool to help structure the review. The content is accurate and based on documented feedback. No external parties are involved.
Is this acceptable, unacceptable, or a gray area?
Scenario 5 of 10
Using AI to explain a complex technical concept to a non-technical stakeholder
A developer is about to present a database migration plan to their company's finance team. They use an AI tool to help generate a plain-language explanation of why the migration is necessary and what the risks are. No sensitive data is included — just a description of the technical approach.
Is this acceptable, unacceptable, or a gray area?
Scenario 6 of 10
Using an AI coding assistant to generate the entire codebase for a new internal tool — without reviewing the output
A developer uses an AI assistant to write a full internal data processing script — about 400 lines. They've been using this AI tool successfully for months and trust its output. Under deadline pressure, they deploy the script to production without doing a code review or testing it against edge cases.
Is this acceptable, unacceptable, or a gray area?
Scenario 7 of 10
Using AI to help interpret a regulation your company must comply with
A compliance analyst uses an AI tool to get a plain-language summary of a new data residency regulation. They use the summary to inform their initial understanding — but then verify the interpretation with their legal team before making any compliance decisions based on it.
Is this acceptable, unacceptable, or a gray area?
Scenario 8 of 10
Using an AI image generator to create visuals for a client presentation using the client's brand colors and logo description
A designer uses an AI image tool to generate custom visuals for a client presentation. They describe the client's brand colors and design aesthetic in the prompt. The client's name appears in the prompt. The AI tool used is not on the company's approved list.
Is this acceptable, unacceptable, or a gray area?
Scenario 9 of 10
Using AI to generate interview questions for a job candidate, including questions about gaps in employment
An HR tech specialist uses an AI tool to generate a structured interview question set. The AI includes several questions about gaps in employment history. The job is in a US-based company.
Is this acceptable, unacceptable, or a gray area?
Scenario 10 of 10
Using AI to generate API documentation for an open-source library your company maintains
A developer uses an AI tool to generate comprehensive documentation for a public, open-source library their company maintains. The library's source code is already publicly available on GitHub. No proprietary or sensitive information is involved.
Is this acceptable, unacceptable, or a gray area?

📌 Article Summary

The Patterns Across All Ten Scenarios

The tool matters as much as the task. Several scenarios (2, 4, 8) involve reasonable tasks made problematic by using the wrong tool — unapproved tools, free consumer tiers for sensitive work. The same task on an approved enterprise tool would often be acceptable.

Unreviewed AI output is the recurring problem. Scenario 6 shows this most clearly — the issue wasn't using AI for code, it was deploying without review. Professional standards for output quality apply regardless of how the output was generated.
